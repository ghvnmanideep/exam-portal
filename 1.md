Examora Core Workflow & Code Documentation
This document explains the core proctoring workflow of your application and breaks down the functionality, state variables (useState), props, event listeners, and essential functions of the four main components: Exam.tsx, CameraRecorder.tsx, ScreenRecorder.tsx, and CameraPopup.tsx.

🏗️ 1. Project Workflow Overview
The examination proctoring system follows this sequence to ensure security:

Initial Entry (Exam.tsx): The student enters the exam page. The component is mounted and a series of "setup steps" begins (start -> camera -> screen -> ready_to_start -> ready). The student is first shown the Rules and Regulations.
Camera Initialization (CameraRecorder.tsx): After agreeing to the rules, the camera setup triggers. The CameraRecorder component is mounted. It requests video/audio permissions, begins capturing frames (every 20s) and audio chunks silently to IndexedDB, and passes the live stream back to Exam.tsx.
Screen Sharing Initialization (ScreenRecorder.tsx): Once the camera is verified, screen sharing setup starts. ScreenRecorder strictly enforces the selection of the "Entire Screen" option. On mobile, this gracefully falls back to html2canvas page capturing. It captures the screen every 10s and saves it to IndexedDB.
Live Proctoring & Exam UI (Exam.tsx & CameraPopup.tsx): The state changes to ready. The exam timer starts, and questions are rendered. The CameraPopup.tsx component receives the camera stream and renders a draggable window so the student knows they are being watched.
Continuous Enforcement (Exam.tsx): Specific useEffect hooks monitor cursor position, tab switching, face detection, keyboard shortcuts (blocking copy/paste/print-screen), right-clicks, and fullscreen status. Rule violations trigger warnings or automatic exam submission.
📝 2. Detailed Component Breakdown
A. Exam.tsx (The Central Hub)
This is the largest component. It handles the exam setup, timer, security event listeners, rendering questions, and calculating the final results.

State Variables (useState)
timeLeft: Manages the 30-minute countdown (1800 seconds).
answers: An object mapping question IDs to the selected option index ({ 1: 0, 2: 3 }).
isFinished: A boolean that toggles the "Results/Grade" screen when the exam ends.
setupStep: Tracks user progress ('start' | 'camera' | 'screen' | 'ready_to_start' | 'ready').
isCameraStarted & isScreenStarted: Booleans controlling when to mount the recorder components.
permissionDenied & screenPermissionDenied: Track if the user rejected browser permissions or selected the wrong screen type.
showRightClickToast & showScreenshotToast: Booleans triggering UI popup notifications on rule violations.
cameraStream: Holds the MediaStream object from the camera, passed down to CameraPopup.
isFullscreen: Tracks if the browser is currently in fullscreen mode.
currentQuestionIndex: Tracks which question the user is currently viewing.
faceWarningCount & showFaceWarning: Tracks how many times the user's face left the frame and toggles the warning modal.
fullscreenTimeLeft: A 30s countdown timer triggered if the user exits fullscreen mode.
isBlurred: Tracks if the window lost focus.
shortcutViolationCount & showViolationWarning: Tracks how many times forbidden keybinds were pressed.
showMobileScreenshotOverlay: Triggers a black screen flash on mobile when a screenshot is attempted.
Core Event Listeners & Effects (useEffect)
This component heavily relies on useEffect to attach global DOM listeners:

Fullscreen Enforcer (fullscreenchange): Detects when the user exits fullscreen. It triggers the fullscreenTimerRef which starts a 30s countdown to auto-submission.
Right-Click & Print Blocker (contextmenu, beforeprint): Calls e.preventDefault() to block right-clicking and printing. Shows a toast notification.
Shortcut & Clipboard Blocker (keydown, copy, cut, paste, keyup): Listens to all keystrokes. Checks for ctrl or meta keys combined with c, v, p, s, f, PrintScreen, etc. It calls e.preventDefault(), clears the clipboard via navigator.clipboard.writeText(''), and logs a violation.
Mobile Protections (touchstart, visibilitychange): Blocks multi-touch, prevents long-press context menus, and flashes a black overlay (showMobileScreenshotOverlay) if the document visibility changes rapidly (indicative of a mobile screenshot).
Background Anti-Cheat (Clipboard clearing & Blur): Runs an interval every 5 seconds clearing the clipboard. Listens to blur and focus to track when the user leaves the window.
Tab Switching (visibilitychange): Logs a violation when document.hidden becomes true. Auto-submits on the 3rd attempt.
Face Detection Manager: Uses a custom hook useFaceDetection. If isFaceDetected becomes false, it triggers setTimeout (4 seconds). If the face doesn't return, it adds a warning strike.
Cursor Tracking (mouseleave, mouseenter): Detects if the mouse leaves the browser window. Starts a 30-second timer (mouseLeaveTimerRef) that auto-submits if the cursor doesn't return.
Key Functions
startCameraSetup() / startScreenSetup(): Triggered by buttons, updates the setupStep to mount the specific recorder component.
startExam(): Calls requestFullscreen() and changes setupStep to ready to reveal the questions.
handleOptionSelect(id, index): Updates the answers state when a radio button is clicked.
finishExam(): Sets isFinishedRef.current = true, ends the timer, exits fullscreen, and saves results to localStorage.
recordViolation(type): Pushes a violation string and timestamp to an array in localStorage.
B. CameraRecorder.tsx
Mounted during setup step 2. Runs silently in the DOM structure (1x1 pixel) to capture the webcam and microphone without disrupting the UI.

Props
onPermissionDenied: Function called if the user blocks camera access.
onStream: Function called with the resolved MediaStream object. It passes the stream back to Exam.tsx.
Core Functions & Hooks
useEffect (on Mount): Instantiates navigator.mediaDevices.getUserMedia(). Assigns the stream to the hidden <video> element. Emits the stream via onStream?.(stream).
Audio Recording Loop (startAudioChunk): Instantiates a MediaRecorder. It listens to ondataavailable and saves chunks of audio to IndexedDB every 20 seconds.
Video Frame Capture (captureImage): Triggers every 20s via setInterval. It draws the current video frame onto an invisible HTML <canvas>, converts it to a base64 JPEG (canvas.toDataURL()), checks if the file size is $>5 \text{ KB}$ (to ensure it isn't a blank frame), and calls saveImage().
DOM Structure: Renders a 1x1 px <div> containing a <video> and a <canvas>. It is kept at full opacity but placed off-screen so browsers do not throttle the video rendering engine.
C. ScreenRecorder.tsx
Mounted during setup step 3. Records the screen and strictly validates the selected screen surface.

Props
onPermissionDenied: Function called if the user blocks screen access or selects a specific window instead of the entire screen.
onStreamStop: Triggered if the user clicks "Stop sharing" on the browser's native floating bar.
onReady: Fires once the strict "Entire Screen" check is passed.
Core Functions & Hooks
useEffect (on Mount):
Checks if the device is a mobile device via navigator.userAgent. If it is mobile, it bypasses the standard getDisplayMedia API and sets up an html2canvas fallback interval that "takes a picture" of the website's DOM every 10 seconds.
On Desktop, it requests navigator.mediaDevices.getDisplayMedia({ video: { displaySurface: 'monitor' } }).
Security Check: Reads videoTrack.getSettings().displaySurface. If it is not exactly 'monitor', it immediately stops the stream, alerts the user, and calls onPermissionDenied().
Screen Capture (captureImage): Similar to the camera recorder, it repeatedly draws the screen stream onto an off-screen <canvas> every 10 seconds and saves the base64 JPEG to IndexedDB.
Event Listeners: Listens to the ended event on the video track to instantly catch users stopping the screen share via native browser controls.
D. CameraPopup.tsx
Rendered when the exam reaches ready_to_start and ready. It provides a draggable live UI so the student knows camera tracking is active.

Props
stream: The identical MediaStream passed down from Exam.tsx (originally generated by CameraRecorder.tsx).
State Variables (useState)
pos: Object { x, y } defining the absolute pixel coordinates of the popup on the screen.
minimized: Boolean toggling whether the video feed is visible or hidden (showing only the header).
Core Functions & Hooks
useEffect (stream connection): Simply connects the stream prop to the videoRef.current.srcObject and plays it.
Drag Mechanics (onMouseDown, onMouseMove, onMouseUp):
onMouseDown: Calculates the initial offset (where the mouse clicked relative to the top-left of the popup) and turns on the dragging.current flag.
onMouseMove (attached to window): Continuously updates setPos({x, y}) using e.clientX and e.clientY, bounds-checking to ensure the popup doesn't get dragged completely off the screen.
onMouseUp: Turns off the dragging.current flag.
